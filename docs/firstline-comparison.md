## 平台定性

**FIRST LINE**（艾魁爾科技股份有限公司，2018 年 11 月創立）：台灣本土企業客服 SaaS，主打「強大但不複雜」，定位中小到中大型企業。核心賣點：多通道統一收件匣 + 無程式碼 Chatbot + 行銷自動化（需加購行銷套件）。

定價結構（2026 年確認）：
- **Mini**：$1,900/月，最多 2 客服，5 管道，無自動化腳本
- **Pro**：$3,900/月，最多 6 客服，10 管道，5 條自動化腳本
- **Enterprise**：$5,900 起/月，無限授權，100 管道，50 條腳本，API 整合
- **行銷套件加購**：$4,500/月（分眾廣播、旅程自動化、優惠券等行銷功能需另購）
- **雲端電話加購**：$3,000/月/客服

## Block 1：FIRST LINE 全功能盤點（35 項）

### 📥 通道整合

1. **LINE Official Account 整合**：支援 LINE Messaging API 完整接入，含文字、圖片、貼圖、影音訊息收發，Webhook 即時接收，多帳號統一管理。

2. **Facebook Messenger 整合**：Facebook 私訊進入統一收件匣，支援 Facebook 訂閱訊息（可突破 24 小時訊息限制）。

3. **Instagram Direct 整合**：IG 私訊統一管理，2024 Q2 確認更新。

4. **網站即時聊天（Live Chat Widget）**：在品牌官網嵌入聊天插件，支援 SHOPLINE、Shopify、一般網站（GTM 部署）。BV SHOP **未見明確文件支援，需確認**。

5. **雲端電話系統（VoIP）**：瀏覽器直接接聽/撥打電話，無需硬體，含三方通話、教練模式（主管監聽）、通話錄音、AI 逐字稿。加購項目，$3,000/月/客服。

6. **電子郵件（Email）管道**：整合 Email 收發至統一收件匣，AI 撰寫功能已支援 Email 管道（2024 Q2 更新）。

7. **WhatsApp 整合**：支援，提供 24 小時未回覆提醒（2024 Q2 新增），但非 Meta BSP 官方認證，API 層級需確認。

8. **Telegram 整合**：首頁提及，細節需確認。

9. **WeChat 整合**：首頁提及，主要對中國市場，細節需確認。

10. **簡訊（SMS）**：首頁提及多通道，具體 SMS 功能需確認。

---

### 💬 對話管理

11. **統一收件匣（Shared Inbox）**：LINE、FB、IG、Email、電話、Live Chat 全部進入同一後台介面，多管道同螢幕管理。

12. **自動話務分配（Auto-Assignment）**：依條件（管道、標籤、客服負載等）自動將對話分配給適合的客服人員或團隊。

13. **對話標籤 / 分類**：對話可加標籤，支援篩選與統計，與工單系統互聯。

14. **轉接（Reassign）**：一鍵移交對話給其他客服，保留完整對話歷史。

15. **內部備忘（Internal Note）**：對話中加入只有內部可見的備註，不傳送給顧客。

16. **快速回覆模板（Canned Replies）**：預設常用回覆語句，客服可快速套用，維持品牌回覆一致性。

17. **多人協同服務**：客服人員可協助處理同事的服務對話（2024 Q2 確認）。

18. **員工狀態時間軸**：可查看客服人員狀態變化紀錄（2024 Q2 新增），支援績效追蹤。

19. **真人客服無縫切換**：Chatbot 流程中設定切換節點，自動轉接真人客服。

20. **交談佇列批次操作**：可批次將多條對話轉入真人隊列或標記完成（2025 Q4 更新）。

---

### 🤖 Chatbot / AI 自動化

21. **無程式碼視覺化流程編輯器**：拖拉式介面設計對話流程，支援 40+ 功能模組，含觸發點、操作、條件判斷節點；最多 13 個快速回覆按鈕；輪播、圖片輪播、互動選單等豐富訊息類型。

22. **AI 自訂代理人（Custom AI Agent）**：自訂 AI 輸入來源、輸出格式，可儲存結果至變數，2025 Q4 支援信件管道。

23. **AI 回答代理人（Answer AI Agent）**：掛載知識庫，自動從文件、PDF、網址擷取問答內容，達到 80% 回答準確率。

24. **AI 判斷代理人（Judgement AI Agent）**：分析訊息意圖（如：退貨意圖、情緒正負向），根據判斷結果自動分流。

25. **AI 對話摘要（AI Summary）**：自動摘要長篇對話，讓接手客服秒懂案件背景。

26. **AI 逐字稿**：針對電話錄音生成完整文本，可作為服務品質訓練素材（2024 更新）。

27. **AI 影像代理人**：辨識圖片內容，應用於發票辨識、名片擷取、產品識別（2025 Q4 更新）。

28. **隨機節點**：在 Chatbot 流程中隨機進入子流程，並可設定各子流程轉入機率（用於 A/B 測試或抽獎模擬）。

29. **圖文選單動態切換節點**：Chatbot 流程中依條件動態指派不同圖文選單，實現個人化互動內容。

30. **知識庫系統（Knowledge Base）**：集中管理內外部知識文件，AI 可從中檢索並直接回答，支援 PDF / TXT / 網址自動提取。

31. **AI 洞察儀表板**：含 AI 交談情況（代理人參與度）、AI 回覆情況（知識庫完整度）、AI 回覆品質（準確度、實用性）三大分析（2025 Q4 新增）。

### 👥 顧客資料 / CRM

32. **顧客 360 檔案**：深度記錄顧客的互動歷史、標籤、自訂欄位、訂單資料，在對話介面側邊即時顯示。

33. **自訂欄位（10 種類型）**：高彈性擴展顧客資料架構，包含文字、數字、日期、選項、階層式選項（2025 新增，支援多階層分類）等欄位類型。

34. **顧客互動度分級**：依互動頻率自動分類（陌生人 / 一般人 / 熟客），可作為行銷分眾依據。

35. **訂單管理 API**：提供完整訂單 CRUD API，含訂單來源、金額、付款狀態（9 種）、履行狀態（9 種）、退貨狀態（10 種），可從外部電商系統同步訂單資料至 FIRST LINE 顧客檔案。電商平台整合：**SHOPLINE** 有文件支援，**Shopify** 有文件支援，**BV SHOP 未見官方文件，需確認是否有私有串接**。

## Block 3：值得吸收的功能清單（TOP 8）

### 1. AI 對話摘要（Auto Summary）

**FIRST LINE 怎麼做**：AI 自動濃縮對話內容，讓接手客服一眼看到案件背景，不用從頭讀訊息。

**我們可以怎麼做**：
- 在對話介面加一個「摘要」按鈕，呼叫 Claude API（已有整合），傳入最近 N 筆訊息
- 回傳摘要存在對話 metadata 欄位，可在轉接時自動帶給接手客服

**工時估計**：1-2 天

**商業價值（梵森場景）**：保養和服飾客戶常有複雜退換貨問題，需要多次往返，接手客服最怕要重讀 50 筆訊息。摘要功能直接解決這個痛點。

---

### 2. 貼文自動回覆（FB/IG 留言觸發私訊）

**FIRST LINE 怎麼做**：指定某篇 FB 或 IG 貼文，當有人留言時系統自動發送私訊，可附帶優惠券或觸發 Chatbot 流程。

**我們可以怎麼做**：
- FB Graph API + IG Webhook 的 `comments` 事件已在我們的整合範圍內
- 後台加設定頁：「指定貼文 ID → 自動回覆訊息 / 觸發腳本」
- 搭配現有廣播系統，留言後可加入分眾名單

**工時估計**：2-3 天

**商業價值（梵森場景）**：梵森在 IG 做活動貼文、新品發布，用留言觸發私訊可以直接將公開互動轉為一對一銷售對話，轉換率遠高於一般廣告。

---

### 3. 優惠券管理（發放 → 核銷全程追蹤）

**FIRST LINE 怎麼做**：可透過 Chatbot 流程或人工發送優惠券，追蹤每張券的開啟、兌換、核銷狀態，活動績效清楚呈現。

**我們可以怎麼做**：
- 後台新增「優惠券」資料表（代碼、折扣方式、使用期限、使用次數限制）
- 在廣播系統加「附上優惠券」選項，發送後記錄個人領取狀態
- 核銷端：客服手動核銷 or LINE LIFF 核銷頁面（掃 QR Code）

**工時估計**：3-5 天（含 LIFF 核銷頁）

**商業價值（梵森場景）**：保養品牌活動（如週年慶、開箱禮）最常用優惠券，現在沒有系統管理，全靠人工記錄，容易漏發漏核。

---

### 4. 追蹤連結（UTM + 點擊自動貼標）

**FIRST LINE 怎麼做**：廣播或訊息中的連結加入 UTM 參數，顧客點擊後自動在其 CRM 資料貼上對應標籤（如：點擊「新品」連結 → 自動貼「對新品有興趣」標籤），2025 年更新支援 LIFF 整合，點擊後可觸發 Chatbot 流程。

**我們可以怎麼做**：
- 建立短連結服務（或用現有 Cloudflare 重定向）
- 點擊時記錄 user_id + 連結ID，觸發 Webhook 自動貼標
- 對接現有廣播分眾系統，讓貼標後立即可用於下次廣播篩選

**工時估計**：2-3 天

**商業價值（梵森場景）**：梵森同時有保養品（聖研）和服飾（梵森），可用追蹤連結分辨哪些顧客對哪條產品線有興趣，廣播時精準分眾。

---

### 5. 顧客互動度自動分級

**FIRST LINE 怎麼做**：依據顧客互動頻率，自動分為「陌生人 / 一般人 / 熟客」三級，可作為行銷分眾與服務優先序的依據。

**我們可以怎麼做**：
- 在顧客資料表加 `engagement_score` 欄位
- 排程任務每日計算：最近 30 天的互動次數、廣播開啟率、主動發訊次數
- 顧客列表頁顯示分級標籤，廣播篩選可選「熟客 only」或「久未互動」

**工時估計**：2 天

**商業價值（梵森場景）**：熟客優先提供 VIP 服務體驗，久未互動者觸發喚醒序列，精準配置客服資源。

---

### 6. Chatbot 流程節點觸發次數統計

**FIRST LINE 怎麼做**：每個 Chatbot 節點都記錄觸發次數，可看到哪個節點最常被點、哪裡是斷點（顧客停止互動的地方），2024 Q2 新增。

**我們可以怎麼做**：
- 現有規則引擎執行時，每次觸發在 `rule_execution_log` 記錄節點 ID
- 報表頁加「流程漏斗圖」：各節點觸發次數 → 視覺化找出斷點

**工時估計**：2 天

**商業價值（梵森場景）**：梵森活動 Chatbot（如抽獎流程、問卷收集）可以清楚知道哪一步顧客放棄，迭代優化轉換率。

---

### 7. 訂單同步 API（BV SHOP 串接）

**FIRST LINE 怎麼做**：提供完整訂單 CRUD API，外部電商可將訂單（含付款狀態、物流狀態、退貨狀態）同步到顧客 CRM 檔案，客服在對話介面可即時看到顧客最新訂單狀態。注意：**FIRST LINE 與 BV SHOP 的官方合作關係無法在公開文件中確認**，可能是透過 API 自行串接。

**我們可以怎麼做**：
- 建立 `orders` 資料表，關聯顧客 ID
- 對外開放 `/api/orders` Webhook 端點，BV SHOP 訂單成立/更新時推送
- 客服介面側邊欄顯示顧客最近 N 筆訂單（訂單號、品項、金額、物流狀態）
- 延伸：顧客查詢訂單時，Chatbot 自動回覆訂單狀態

**工時估計**：3-5 天（含客服介面展示）

**商業價值（梵森場景）**：梵森用 BV SHOP，顧客在 LINE 問「我的訂單在哪」，客服現在要手動去 BV 後台查，有了訂單同步後客服介面直接顯示，效率倍增。對老闆來說這是**最直接的痛點解方**。

---

### 8. 工單系統（Ticketing）輕量版

**FIRST LINE 怎麼做**：工單可設定分類、優先度（低/中/高/緊急）、自訂欄位，支援跨部門追蹤，可與顧客 CRM 互聯，工單自動化觸發規則。

**我們可以怎麼做**：
- 在現有對話系統加「建立工單」按鈕，工單連結對話 ID 和顧客 ID
- 輕量版：分類 + 優先度 + 指派人 + 狀態（開啟/處理中/結案）
- 不做複雜的跨部門流程，只做梵森實際用得到的

**工時估計**：3-5 天（輕量版）

**商業價值（梵森場景）**：退貨、換貨、商品瑕疵這類需要追蹤進度的案件，現在對話結束就不見了，工單讓這些案件有明確的 owner 和截止期限。

## Block 4：戰略建議

### 必做的 3 個（馬上動手）

1. **BV SHOP 訂單同步 API**（功能 #7）
   老闆的核心痛點：客服查訂單要跳到 BV 後台，一天要查幾十次。做了以後客服效率立竿見影，也是梵森「自建客服中心」相對於 FIRST LINE 最直接的價值驗證。預計 3-5 天，商業價值最高。

2. **貼文自動回覆（IG/FB 留言觸發）**（功能 #2）
   梵森在社群最常做的就是新品貼文和活動貼文，留言往往幾十上百條，人工一一回覆效率極低。這個功能成本低（2-3 天），但日常使用頻率極高。

3. **AI 對話摘要**（功能 #1）
   已有 Claude API 整合，幾乎是純 prompt 工程 + 一個 UI 按鈕，1-2 天就能做完。對有複雜退換貨案件的保養品牌（聖研）客服幫助最直接。

---

### 可做但不急的 3 個

1. **優惠券管理**（功能 #3）：目前梵森的活動頻率和規模，人工管理還撐得住。等到活動量上來再做，工時 3-5 天，有明確需求再啟動。

2. **顧客互動度分級**（功能 #5）：對分眾行銷有長期價值，但需要先累積一段時間的互動資料才能有意義。建議等廣播系統跑 3 個月後再開這個功能。

3. **Chatbot 節點數據統計**（功能 #6）：有用，但要先有足夠的 Chatbot 流程在跑才有數據可看。目前先把流程做好，再加數據追蹤。

---

### 建議跳過的 3 個

1. **雲端電話系統（VoIP）**：FIRST LINE 要加購 $3,000/月/客服，需要 SIP/電信基礎設施。梵森的客服溝通管道以 LINE/IG 為主，電話需求很低，投入產出不符。

2. **WhatsApp 整合**：台灣 WhatsApp 滲透率低，梵森客群是台灣本地保養/服飾消費者，不需要。WhatsApp Business API 還有授權費用和 template message 費用，直接跳過。

3. **視覺化 Chatbot 流程編輯器（拖拉式）**：FIRST LINE 的核心差異化之一，但我們的目標用戶（梵森自用）不是 no-code 用戶，老闆有技術支援（就是我們）。做一個完整的 Flow Builder 要 2-4 週，C/P 值低。現有腳本設定介面夠用，把時間投在其他更有感的功能。

---

### 總結評分

我們對標 FIRST LINE 的整體分數是 **7 / 10**。

核心客服功能（多通道收件匣、自動分配、廣播分眾、顧客旅程、AI 草擬）我們已有對等甚至更強的實作。主要差距在：電商訂單整合（BV SHOP 同步）、完整優惠券管理、社群留言觸發、AI 摘要這幾個「提升每日使用體驗」的功能。FIRST LINE 行銷套件功能豐富但需加購 $4,500/月，我們把廣播分眾內建免費是明顯優勢。我們的多業主 SaaS 架構、品牌 DNA × AI 草擬、抽獎遊戲模組是 FIRST LINE 沒有的差異化。
